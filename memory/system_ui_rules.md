# Mission Control UI 设计规则

## 目标
持续优化 Mission Control 网站的界面设计与用户体验，使其达到现代专业控制台级别（类似 Vercel、Palantir、Linear 风格）。

## 长期规则

### 1. 网站必须具备现代控制台风格
- **深色主题优先**：专业、沉浸、低疲劳
- **卡片式布局**：信息模块化，边界清晰
- **清晰层级结构**：视觉权重合理，重点突出
- **响应式设计**：适配桌面、平板、移动端

### 2. 每次迭代必须优化至少一项
- UI 美观度（配色、字体、间距）
- 信息层级结构（布局、分组、优先级）
- 数据可视化（图表、状态指示、趋势）
- 交互体验（动画、反馈、操作流畅度）

### 3. 优先优化以下组件
1. **Dashboard 总览页** - 核心数据一目了然
2. **Tasks 页面** - 任务状态清晰可控
3. **Memory 页面** - 记忆检索高效便捷
4. **状态监控模块** - 实时数据可视化

### 4. 所有改动必须
- 保持功能稳定
- 不破坏现有数据
- 可回滚（Git 版本控制）

### 5. 每次优化必须记录
- 更新 `memory/ui_changelog.md`
- 记录改动内容、原因、效果

## 设计参考
- **Vercel**：简洁、现代、开发者友好
- **Palantir**：数据密集、专业、功能强大
- **Linear**：流畅、精致、极致体验

## 当前状态
- 基础框架：Next.js + Tailwind CSS
- 当前风格：黑金配色（#0A0A0A + #FFD700）
- 部署地址：https://mission-control-sigma-six.vercel.app/
