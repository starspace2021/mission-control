# 🔄 自我迭代机制

**系统**: OpenClaw 任务自我优化框架  
**创建时间**: 2026-02-23  
**版本**: v1.0

---

## 📋 迭代流程

```
执行任务 → 记录日志 → 质量评估 → 识别问题 → 生成改进方案 → 更新规则 → 下次执行
```

---

## 📝 执行日志规范

每次任务完成后必须记录：

| 字段 | 说明 | 示例 |
|------|------|------|
| `task_id` | 任务标识 | africa-osint-daily-summary |
| `execution_time` | 执行时间戳 | 2026-02-23T07:00:00+08:00 |
| `duration_seconds` | 执行耗时 | 142 |
| `status` | 成功/失败/部分成功 | success |
| `output_quality` | 输出质量评分 (1-10) | 8 |
| `issues` | 遇到的问题列表 | ["数据源延迟", "关键词误报"] |
| `improvements` | 改进想法 | ["增加备用数据源", "优化过滤规则"] |
| `data_sources` | 使用的数据源 | ["google_news", "reuters"] |
| `keywords_hit` | 关键词命中率 | 0.75 |

## 🔧 迭代执行计划

### 每日迭代 (Daily)

| 时间 | 任务 | 动作 |
|------|------|------|
| 06:30 | 质量评估 | 评估昨日所有任务执行质量 |
| 07:30 | 改进报告 | 生成昨日改进建议 |
| 08:00 | 规则更新 | 自动更新任务配置 |

### 每周迭代 (Weekly)

| 时间 | 任务 | 动作 |
|------|------|------|
| 周日 20:00 | 周度回顾 | 汇总本周所有任务表现 |
| 周日 21:00 | 趋势分析 | 识别长期问题和改进机会 |
| 周日 22:00 | 规则优化 | 更新 SKILL.md 和配置文件 |

### 每月迭代 (Monthly)

| 时间 | 任务 | 动作 |
|------|------|------|
| 每月1日 | 月度总结 | 生成本月迭代报告 |
| 每月1日 | 架构审查 | 评估整体系统健康度 |
| 每月1日 | 大版本更新 | 实施重大改进 |

## 📊 质量评分标准

### 输出质量评分 (1-10)

| 分数 | 等级 | 说明 |
|------|------|------|
| 10 | 完美 | 超出预期，可作为模板 |
| 8-9 | 优秀 | 符合标准，有小改进空间 |
| 6-7 | 合格 | 基本达标，需要优化 |
| 4-5 | 较差 | 有明显问题，需立即改进 |
| 1-3 | 失败 | 无法使用，需重新设计 |

### 问题严重程度

| 级别 | 说明 | 响应时间 |
|------|------|----------|
| P0 | 系统故障，任务无法执行 | 立即 |
| P1 | 严重问题，输出质量大幅下降 | 4小时内 |
| P2 | 中等问题，影响效率或准确性 | 24小时内 |
| P3 | 轻微问题，有改进空间 | 一周内 |

## 🚀 快速开始

### 启用自我迭代的任务

1. **编辑任务配置**，添加 `self_iterate: true`
2. **配置评估维度**，指定质量指标
3. **设置迭代频率**，daily/weekly/monthly
4. **系统自动执行**，无需人工干预

### 查看迭代报告

```
读取 memory/iterations/daily/YYYY-MM-DD.json 获取每日报告
读取 memory/iterations/weekly/YYYY-WXX.json 获取每周报告
```
